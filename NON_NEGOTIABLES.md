# iThoddoo Maldives Non-Negotiables

Project codename: Project Atlas

These principles guide all future product, design, and engineering decisions.

## Principles

- Mobile-first experience.
- Fast loading pages.
- Reusable components only.
- No duplicated UI.
- Real photography whenever possible.
- Honest and accurate travel information.
- Every feature must help travelers, partners, or the business.
- Quality before quantity.
- Build for scalability, not one-time fixes.
- Build for 8 islands in the future, not just Thoddoo.
- Protect user trust.
- Keep booking simple.
- Keep WhatsApp support fast and visible.
- Use clean TypeScript.
- Use Tailwind consistently.
- Maintain strong documentation.

## Decision Filter

Every new idea should answer:

- Does it help travelers?
- Does it help local businesses?
- Does it improve trust?
- Does it improve booking conversion?
- Does it support future island expansion?

## Project Motto

"Make every visitor feel like they have a trusted local friend in the Maldives."
